# MemeSeek 项目演示流程

这是一份适合面试或项目展示的 1～2 分钟演示脚本。重点是展示一条完整链路，不需要把每个页面都讲一遍。

## 演示前准备

1. 启动 PostgreSQL、后端和前端。
2. 打开 `http://localhost:5173`。
3. 在 AI 设置页确认分析 AI 的 Base URL、模型和 API Key 已配置。
4. 准备一张可以复制到剪贴板的梗图。
5. 确认列表中已有一到两张示例梗图，便于展示搜索和详情。

## 90 秒主流程

### 0～10 秒：介绍产品

可以这样说：

> 这是 MemeSeek，一个个人梗图管理工具。它可以保存梗图，并通过视觉模型自动生成标题、描述、标签和图片文字，之后可以直接搜索和管理。

展示梗图列表页，指出搜索框、卡片和上传入口。

### 10～30 秒：演示 Ctrl+V 上传

1. 点击“上传梗图”打开上传抽屉。
2. 复制一张图片，在上传区域按 `Ctrl+V`。
3. 展示图片预览和文件名。
4. 说明这里也支持传统的点击选择和拖拽上传。
5. 点击“开始上传”。

推荐讲解：

> 粘贴图片不会直接触发 AI 请求，而是先进入统一的预览状态。用户确认后，前端把它作为标准 `File` 走原来的 multipart 上传流程，所以三种图片来源复用同一套校验和状态管理。

### 30～55 秒：展示 AI 分析结果

等待分析状态从 `PROCESSING` 变为 `COMPLETED`，打开新梗图详情。

指出：

- AI 生成的标题。
- 描述和使用场景。
- 推荐标签。
- OCR 识别出的图片文字。

推荐讲解：

> 上传和图片分析是两个清晰的阶段。后端先保存图片，再调用视觉模型；分析结果经过 JSON 校验后写入数据库，失败时会保留失败状态并提供重新分析入口。

### 55～75 秒：演示编辑和搜索

1. 点击“编辑信息”，修改标题或描述。
2. 保存后回到列表。
3. 在搜索框输入标题、标签或 OCR 中的关键词。
4. 展示 URL 同步出现 `?q=`。

推荐讲解：

> 列表数据通过 TanStack Query 管理，编辑、上传和删除成功后会刷新查询缓存。搜索条件同步到 URL，因此页面刷新后仍然可以复现当前搜索。

### 75～90 秒：说明架构和安全边界

打开 AI 设置页，简单指出分析 AI 和内容 AI 的分开配置。

推荐讲解：

> 当前分析 AI 负责图片理解和 OCR，内容 AI 为后续文本改写和自然语言搜索预留。项目是个人本地工具，API Key 暂存在浏览器 localStorage，不写入数据库；如果公开部署，需要改成后端密钥管理并限制 Base URL。

## 面试官追问时的补充点

### 为什么粘贴图片不需要新后端接口？

因为浏览器剪贴板中的图片可以读取成 Blob，再转换成标准 `File`。后端已经支持 multipart 文件上传，所以粘贴、拖拽和点击选择最终走同一个接口。

### AI 失败怎么办？

梗图记录会进入 `FAILED` 状态，界面展示错误信息，用户可以在详情弹窗中重新分析，不需要重新上传图片。

### 为什么先保存图片再调用 AI？

这样可以把文件上传失败和 AI 分析失败区分开，也能在 AI 暂时不可用时保留原始图片，之后重新分析。

### 项目还有哪些可以扩展？

- 批量上传和批量分析。
- 自然语言搜索。
- 内容 AI 改写梗图文案。
- 对象存储替代本地文件保存。
- 后端 API Key 管理、用户登录和权限控制。

## 演示结束语

> 这个项目目前重点完成了从图片上传、AI 分析到搜索和编辑的完整闭环，同时保留了内容生成、批量处理和公开部署的扩展空间。
